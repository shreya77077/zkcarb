# ZKCarb

**Privacy-Preserving Carbon Credit Minting on Hedera**

ZKCarb is a decentralized application that enables verifiable carbon credit issuance without exposing sensitive environmental sensor data on-chain. Using zero-knowledge proofs (Groth16 via snarkjs and circom), the system allows project developers to prove their MRV (Measurement, Reporting, and Verification) data meets certification thresholds and mint a carbon credit on the Hedera testnet — without ever revealing the raw sensor values.

---

## Problem Statement

Carbon credit markets rely on MRV data: CO2 sequestration rates, biomass density, and project area measurements collected from physical sensors. In existing systems, this data is either submitted to a centralized verifier (trust-dependent, single point of failure) or published on-chain directly (exposing proprietary or competitive data publicly). Neither model is satisfactory. Project developers need a way to prove compliance with certification thresholds without disclosing the underlying measurements.

---

## Solution

ZKCarb uses a Groth16 zero-knowledge proof circuit (`mrv_verifier.circom`) to generate a cryptographic proof that the MRV inputs satisfy all threshold constraints. A Poseidon hash of the inputs is computed as a commitment and included as a public signal. Only the proof and commitment — not the raw inputs — are submitted on-chain. A Solidity smart contract deployed on Hedera EVM verifies the proof and, upon success, mints one carbon credit to the caller's address. The commitment is stored on-chain as a nullifier, preventing the same dataset from minting more than one credit.

The result: a trustless, privacy-preserving carbon certification pipeline where verification is fully on-chain and sensor data never leaves the user's device.

## Architecture

```
User Device
└── MRV Inputs (CO2, Biomass, Area)  [private — never sent on-chain]
      └── Poseidon Commitment Hash
      └── Circom Circuit: mrv_verifier.circom
            └── Groth16 Proof (snarkjs)
                  └── Express Proof Server (port 3001)
                        └── Hedera EVM — MRVVerifier.sol + CarbonCredit.sol
                              └── Verify proof → Mint credit
```

**Directory structure:**

```
zkcarb/
├── circuits/
│   ├── mrv_verifier.circom       # ZK circuit with MRV threshold constraints
│   └── build/                    # Compiled wasm, zkey, and verification key
├── contracts/
│   ├── MRVVerifier.sol           # Groth16 verifier (generated by snarkjs)
│   └── CarbonCredit.sol          # Credit minting contract with nullifier check
├── scripts/
│   ├── deploy.ts                 # Hardhat deployment to Hedera testnet
│   ├── mint.ts                   # CLI minting script
│   ├── retire.ts                 # CLI credit retirement script
│   ├── generate_proof.js         # Standalone proof generation
│   ├── generate_commitment.js    # Poseidon commitment utility
│   └── hcs_submit.js             # Hedera Consensus Service logging
├── server/
│   └── index.js                  # Express proof generation server (port 3001)
├── frontend/
│   └── src/
│       └── App.tsx               # React UI — MRV input, proof generation, mint flow
└── hardhat.config.ts
```

## Setup and Installation

### 1. Clone the repository

```bash
git clone https://github.com/shreya77077/zkcarb.git
cd zkcarb
```

### 2. Install dependencies

```bash
# Root dependencies (Hardhat, ethers, snarkjs)
npm install

# Proof server dependencies
cd server && npm install && cd ..

# Frontend dependencies
cd frontend && npm install && cd ..
```

### 3. Configure environment variables

Create a `.env` file in the project root:

```env
HEDERA_EVM_PRIVATE_KEY=your_evm_private_key_from_metamask
```

> To export your EVM private key: MetaMask → Account Details → Export Private Key. This key is used only for contract deployment. Never commit this file.

### 4. Compile the ZK circuit

```bash
# Compile the circuit
circom circuits/mrv_verifier.circom --r1cs --wasm --sym -o circuits/build/

# Generate Powers of Tau
snarkjs powersoftau new bn128 16 circuits/build/pot16_0000.ptau -v
snarkjs powersoftau prepare phase2 circuits/build/pot16_0000.ptau circuits/build/pot16_final.ptau -v

# Groth16 trusted setup
snarkjs groth16 setup circuits/build/mrv_verifier.r1cs circuits/build/pot16_final.ptau circuits/build/mrv_verifier_0000.zkey
snarkjs zkey contribute circuits/build/mrv_verifier_0000.zkey circuits/build/mrv_verifier_final.zkey --name="contribution" -v
snarkjs zkey export verificationkey circuits/build/mrv_verifier_final.zkey circuits/build/verification_key.json
```

### 5. Export the Solidity verifier

```bash
snarkjs zkey export solidityverifier circuits/build/mrv_verifier_final.zkey contracts/MRVVerifier.sol
```

### 6. Compile and deploy contracts

```bash
npx hardhat compile
npx hardhat run scripts/deploy.ts --network hedera_testnet
```

Copy the deployed contract address from the output and update it in `frontend/src/App.tsx`.

### 7. Start the proof generation server

```bash
cd server
node index.js
# Proof server running on http://localhost:3001
```

### 8. Start the frontend

```bash
cd frontend
npm run dev
# App running on http://localhost:5173
```

## Usage

1. Open `http://localhost:5173` in your browser.
2. Click **Connect MetaMask**. Confirm adding Hedera Testnet (Chain ID 296) if prompted.
3. Ensure your MetaMask wallet has testnet HBAR. Get free HBAR at [portal.hedera.com/faucet](https://portal.hedera.com/faucet) using your MetaMask address.
4. Enter MRV values that satisfy the thresholds:
   - CO2 Sequestration >= 1000 kg/ha/year
   - Biomass Density >= 70 tonnes/ha
   - Project Area >= 100 hectares
5. Click **Generate Proof & Mint Credit**.
6. Approve the transaction in MetaMask (gas approximately 0.3 HBAR).
7. On success, the Credits Balance counter increments and a transaction hash is displayed. Click **View on HashScan** to verify the transaction on-chain.

> Each unique combination of inputs can only mint one credit. Resubmitting identical values will revert with `Commitment already exists`.

## Smart Contracts

| Contract           | Purpose                                                    |
| ------------------ | ---------------------------------------------------------- |
| `MRVVerifier.sol`  | Groth16 proof verifier, auto-generated by snarkjs          |
| `CarbonCredit.sol` | Minting logic, nullifier registry, credit balance tracking |

Key functions in `CarbonCredit.sol`:

- `mintCredit(proof, publicSignals)` — verifies the Groth16 proof and mints one credit; reverts on duplicate commitment
- `retireCredit()` — burns a credit to prevent double-counting, consistent with voluntary carbon market standards

## Privacy Model

The Circom circuit takes the following private inputs:

```
co2_reading, biomass_density, area_ha
```

And the following public inputs:

```
co2_threshold (1000), biomass_threshold (70), area_threshold (100), commitment
```

The circuit proves the private values satisfy their respective thresholds without revealing them. The commitment is a Poseidon hash of the three private inputs, used on-chain solely as a nullifier to enforce single-use per dataset.
